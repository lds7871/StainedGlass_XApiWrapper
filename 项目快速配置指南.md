# 🚀 StainedGlass_XApiWrapper 项目快速配置指南
## 配置流程(至少需要的必要流程)

- **可访问地址**：需要内网穿透或云服务器（即可被公网访问的网址）
- **Nginx**: 按照./Nginx配置里的文件覆盖或合并Nginx，注意还需要SSL证书
- **网络代理**：去.src/main/resources的config.properties配置
- **X开发平台APP创建**：访问 [Twitter Developer Portal](https://developer.twitter.com/en/portal/dashboard)创建 App。获取 Client ID 和 Client Secret。配置回调地址（http://xxx.xxx.xxx.xx/callback/twitter/oauth ）。
- **创建数据表**：.src/main/resources下有建表文件，MySQL8
- **配置resources**：复制.yml.properties的example。并启动“B快速配置.cmd”进行配置
- **Maven**：下载Mvn并点击编译和启动两个cmd
- **认证**：调用/callback/twitter/authorize->打开响应网页->点击授权->自动执行/callback/twitter/oauth->授权成功页面返回Json

## 环境配置

### 需要准备
- **操作系统**: Windows / Linux / macOS（我不确定）
- **Java**: JDK 17+
- **Maven**: 3.8.x / 3.9.x
- **MySQL**: 8.0+
- **Nginx**: 1.20+ (用于反向代理)(还需要自建或导入SSL证书)
- **公网地址**：需要本地网络穿透或云服务器(OAuth2.0认证必须)

###  复制示例文件
- cp src/main/resources/application.yml.example src/main/resources/application.yml
- cp src/main/resources/config.properties.example src/main/resources/config.properties

###  OAuth2.0
X开发平台需要OAuth2.0认证，所以必须要配置公网回调地址（yml中callback-url），此外S256和PECK不用手动配置

## Token 管理

### Token 有效期
- **访问 Token**: 2 小时
- **刷新触发**: 剩余时间 < 30 分钟自动刷新
- **自动维护**: ✅ 系统后台自动处理

### Token 存储
- **数据库表**: `twitter_token`
- **存储内容**: 
  - `user_id` - Twitter 用户 ID
  - `access_token` - 访问令牌
  - `refresh_token` - 刷新令牌
  - `expires_at` - 过期时间
  - `created_at` - 创建时间
  - `updated_at` - 更新时间
